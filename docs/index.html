i18n-for-react

i18n-for-react

Modern translation module for React.

Install

npm i i18n-for-react
# or
yarn add i18n-for-react

API

Module exposes next API:

export default globalConfig;
export {
    IConfig,
    ILocales,
    IFallbacks,
    IUnknownPhraseListener,
    IProcessor,
    IParams,
    IPluralParams,
    pluralIntervalProcessor,
    mustacheProcessor,
    I18nContext,
    I18nProvider,
    II18nProviderConfig,
    withI18n,
    rprintf,
    __,
    __mf,
    __n,
    __m
};

Description of this methods you can find in Documentation.

Basic API is same as in i18n-for-browser module.

I18nProvider

Configurator and provider of i18n instance.

Root context configuration example:

<I18nProvider
    locale='en'
    locales={{
        en: {/* ... */},
        ru: {/* ... */}
    }}
>
    {__`hi`}
</I18nProvider>

Fork context example:

<I18nProvider
    locale='en'
    locales={{
        en: {/* ... */},
        ru: {/* ... */}
    }}
>
    {__`hi`}
    <I18nProvider
        locales={{
            en: {/* ... */},
            ru: {/* ... */}
        }}
    >
        {__`goodby`}
    </I18nProvider>
</I18nProvider>

withI18n

Decorator to configure provider of i18n instance.

Example:

@withI18n({
    en: {/* ... */},
    ru: {/* ... */}
})
class Container extends Component {

    state = {
        locale: 'en'
    };

    render() {
        return (
            <I18nProvider
                locale={this.state.locale}
            >
                {__`hi`}
                <I18nProvider
                    locales={{
                        en: {/* ... */},
                        ru: {/* ... */}
                    }}
                >
                    {__`goodby`}
                </I18nProvider>
            </I18nProvider>
        );
    }
}

rprintf()

Every translate function support React-formatted output:

__('Hi, <>%s</>!', 'username', [<b/>])
// or
__('Hi, <>%s</>!', 'username', [_ => <b>{_}</b>])
// will produce:
[
    'Hi, ',
    <b>username</b>,
    '!'
]
